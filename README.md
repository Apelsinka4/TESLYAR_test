# TESLYAR Amazon EU Dashboard

Статичний дашборд для аналізу Amazon P&L/PPC CSV. Менеджер відкриває сторінку, вибирає CSV-файли за потрібні місяці, а дашборд будується прямо у браузері. Дані не відправляються на сервер, якщо не налаштований окремий Claude proxy для summary.

## Як користуватись

1. Відкрийте сайт або локальний `index.html`.
2. Натисніть **Вибрати CSV**.
3. Оберіть P&L-файли та PPC-файли мінімум за два місяці.
4. Виберіть поточний місяць і місяць для порівняння.
5. За потреби натисніть **Оновити через Claude**, якщо налаштований proxy endpoint.

Для локального перегляду через сервер:

```bash
python -m http.server 8000
```

Після цього відкрийте [http://localhost:8000](http://localhost:8000).

## Що зроблено

- Огляд портфелю: Sales, Net profit, Margin з MoM-порівнянням.
- P&L портфелю по країнах: сортування за зміною продажів, прибутку, маржі або PPC ACOS.
- Top-N по артикулах: продажі, прибуток, зростання прибутку або рейтинг уваги.
- Аналітичне резюме: локальне summary + опційне оновлення через Claude API.
- Багаторазове оновлення: можна завантажувати новий набір CSV без зміни коду.

## Claude API

GitHub Pages не може безпечно зберігати `ANTHROPIC_API_KEY`, тому прямий виклик Claude з фронтенду не використовується. Для Claude summary потрібен маленький proxy, наприклад Cloudflare Worker з файлу `workers/claude-summary-worker.js`.

1. Створіть Cloudflare Worker.
2. Додайте secret `ANTHROPIC_API_KEY`.
3. Задеплойте код з `workers/claude-summary-worker.js`.
4. У `config.js` вставте URL worker:

```js
window.CLAUDE_PROXY_URL = "https://your-worker.your-subdomain.workers.dev";
```

Worker використовує Anthropic Messages API і легку модель `claude-haiku-4-5-20251001`.

## Стек і чому так

Стек: vanilla HTML/CSS/JS без бекенду для основного дашборда. Це простий шлях для нетехнічного користувача: достатньо відкрити сторінку і вибрати файли. CSV парсяться у браузері, тому реальні фінансові дані не треба публікувати в репозиторій.

Код обробляє `;` у P&L-файлах, європейську нотацію чисел з комою та пробілами, різний регістр і подвійні підкреслення у назвах файлів. PPC-звіти підтягуються на рівні країни для ACOS/ROAS.
